Hello World! - A First Look

We will begin our exploration of JActor2 with a program, HelloWorld, that prints "Hello World!". In the process, we will encounter a number of classes which play a major role when using JActor. 

import org.agilewiki.jactor2.core.blades.misc.Printer;
import org.agilewiki.jactor2.core.plant.BasicPlant;
import org.agilewiki.jactor2.core.plant.Plant;
import org.agilewiki.jactor2.core.messages.SyncRequest;

public class HelloWorld {
    public static void main(final String[] _args) throws Exception {
        BasicPlant plant = new Plant();
        try {
            Printer.printlnAReq("Hello World!").call();
        } finally {
            plant.close();
        }
    }
}

Every program must have one Plant and a program will not normally exit until its Plant is closed.

BasicPlant is an interface implemented by Plant and defines its API, excluding only the methods to support Modular Programming.

Printer 

            Printer.printlnAReq("Hello World!").call();

A printer is a BlockingBlade, which is to say an application object that uses a BlockingReactor. It prints to a PrintStream, System.out being the default.

The static method Printer.printlnAReq returns an AsyncRequest which, when called, prints the string. And if there is no printer, one is safely created.

The call method invokes the request to print the string. But it is printed from the printer's own thread, not from the main thread.

The call method of a request provides one way of invoking a request. The printRequest.call method sends the request to the printer's thread and blocks the main thread until a result is received. The call method is the only method in JActor which blocks a thread. (The call method can not be used by a blade, as it must not be invoked from a thread in the plant's thread pool.)
