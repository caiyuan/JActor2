SpeedReport - External Requests

SpeedReport is a SyncRequest that prints a brief report that is useful when writing benchmarks. The report is generated entirely on the same thread used by Printer. 

import org.agilewiki.jactor2.core.blades.misc.Printer;
import org.agilewiki.jactor2.core.blades.misc.SyncPrinterRequest;
import org.agilewiki.jactor2.core.plant.Plant;
import org.agilewiki.jactor2.core.messages.AsyncRequest;
import org.agilewiki.jactor2.core.messages.AsyncResponseProcessor;

public class SpeedReport extends SyncPrinterRequest {
    
    public static AsyncRequest<Void> startAReq(
            final String _heading, 
            final long _ns, 
            final long _count) {
        return new AsyncRequest<Void>(Plant.getSingleton().getReactor()) {
            AsyncResponseProcessor dis = this;

            @Override
            public void processAsyncRequest() throws Exception {
                send(Printer.stdoutAReq(), new AsyncResponseProcessor<Printer>() {
                    public void processAsyncResponse(final Printer _printer) throws Exception {
                        SpeedReport sr = new SpeedReport(_printer, _heading, _ns, _count);
                        send(sr, dis);
                    }
                });
            }
        };
    }

    private final String heading;
    private final long ns;
    private final long count;
    
    private SpeedReport(final Printer _printer, 
            final String _heading, 
            final long _ns, 
            final long _count) throws Exception {
        super(_printer);
        heading = _heading;
        ns = _ns;
        count = _count;
    }

    @Override
    public Void processSyncRequest() throws Exception {
        println("");
        println(heading);
        printf("Test duration in nanoseconds: %,d%n", ns);
        printf("Number of exchanges: %,d%n", count);
        if (ns > 0)
            printf("Exchanges per second: %,d%n%n", 1000000000L * count / ns);
        return null;
    }
}

SyncPrinterRequest 

public class SpeedReport extends SyncPrinterRequest {
    ...
}

SyncPrinterRequest extends SyncRequest<Void>. SyncPrinterRequest accesses the public resources of Printer locally, without recourse to messages, as it operates as an extension of Printer. Local access to Printer means that SyncPrinterRequest can easily implement protected methods, i.e. println and printf, for working with the Printer blade.

Processing a SpeedReport Request 

    @Override
    public Void processSyncRequest() throws Exception {

        return null;

When processing a SpeedReport request, the println and printf methods inherited from SyncPrinterRequest can be used directly.

AsyncRequest 

    public static AsyncRequest<Void> startAReq(
            final String _heading, 
            final long _ns, 
            final long _count) {
        return new AsyncRequest<Void>(Plant.getSingleton().getReactor()) {
            ...
        };
    }

Async requests are similar to sync requests, only the response must be passed back via the processAsyncMethod rather than with a return statement. And the local method does not apply to AsyncRequest. Also you can send requests that are tied to other reactors in an AsyncRequest but you can not do that within a SyncRequest.

Creating an AsyncRequest with the reactor of the plant means that it can access the public methods of Printer directly, without recourse to messages, as it uses the same reactor as Printer and consequently operates on the same thread.

AsyncResponseProcessor 

            AsyncResponseProcessor dis = this;

Subclasses of AsyncResponseProcessor are used to implement the logic that will be used to process a response. So when you use a send, you typically instantiate an anonymous subclass with a processAsyncResponse method. AsyncRequest also implements the AsyncResponseProcessor to provide an easy means of passing back a result.

processAsyncRequest 

            @Override
            public void processAsyncRequest() throws Exception {
                ...
            }

The processAsyncRequest method is called when the request is to be processed. It is similar to SyncRequest.processSyncRequest, except that processAsyncRequest has no return value.

stdoutAReq 

                send(Printer.stdoutAReq(), new AsyncResponseProcessor<Printer>() {
                    public void processAsyncResponse(final Printer _printer) throws Exception {
                        ...
                    }
                });

The stdoutAReq method is an asynchronous request to return the facility's printer, safely creating and registering a printer if there was none.

send 

                SpeedReport sr = new SpeedReport(_printer, _heading, _ns, _count);
                send(sr, dis);

The send method invokes a request and provides a callback for processing the response. In this case, the response is sent back to the invoker of request returned by startAReq. The alternative is usually to create a subclass of AsyncResponseProcessor, but that would be pretty redundant in this case: 

                send(sr, new AsyncResponseProcessor<Void>() {
                    @Override
                    public void processAsyncResponse(final Void _response) throws Exception {
                        dis.processAsyncResponse(null);
                    }
                });

Test

We will use the SpeedReportValidator program to test SpeedReport: 

import org.agilewiki.jactor2.core.blades.misc.Printer;
import org.agilewiki.jactor2.core.facilities.BasicPlant;
import org.agilewiki.jactor2.core.facilities.Plant;

public class SpeedReportValidator {
    public static void main(final String[] _args) throws Exception {
        BasicPlant plant = new Plant();
        try {
            SpeedReport.startAReq("Speed Report Validation", 1000L, 1L).call();
        } finally {
            plant.close();
        }
    }
}
