Getting Started - Jars and Scripts

You will need a few things to work with JActor2 core:

  • JDK 7 or later.
  • slf4j.
  • Google pcollections.
  • Google guava.
  • The JActor2 core jar file. And
  • A test script. For windows, you can use test.bat: 
    del *.class
javac -classpath jactor2-core-0.7.0.jar;slf4j-api-1.7.5.jar *.java
java -classpath jactor2-core-0.7.0.jar;slf4j-api-1.7.5.jar;slf4j-simple-1.7.5.jar;pcollections-2.1.2.jar;guava-15.0.jar;. %1
    Othrwise, you can likely use test.sh: 
    rm -fr *.class
javac -cp jactor2-core-0.7.0.jar:slf4j-api-1.7.5.jar *.java
java -cp jactor2-core-0.7.0.jar:slf4j-api-1.7.5.jar:slf4j-simple-1.7.5.jar:pcollections-2.1.2.jar:guava-15.0.jar:. $1

    • Debug

    JActor has a debug mode. And when running in debug mode, additional checking is done that is likely too slow for production use. But it can be great when testing. Debug mode is set by assigning "true" to the jactor.debug system property. For example, you can turn debug mode on by adding this parameter when running JActor: 

    -Djactor.debug=true

    One of the things that jactor does when in debug mode is to track all pending requests. They are kept in the pendingRequests variable in each Facility instance. 

        /**
     * When DEBUG, pendingRequests holds the active requests ordered by timestamp.
     */
    public final ConcurrentSkipListMap> pendingRequests =
            Plant.DEBUG ? new ConcurrentSkipListMap>() : null;

    One fairly common bug is when no response is passed back for a request. This can cause an application to hang unexpectedly. Difficult to debug without some means of finding the pending requests. Fortunately, the presence of this type of bug is quickly discovered in unit testing.

    Configuration

    When no other configuration is specified, the PlantConfiguration PlantConfiguration class is used.

    This timer is set to a default value of 1 second, though it is easily customized for any part of JActor. The default can also be changed by assigning a value, in milliseconds, to the system property jactor.threadInterruptMilliseconds. For example, you can set the timer to 1 minute by adding this parameter when running JActor: 

    -Djactor.threadInterruptMilliseconds=60000

    If, for example, you have a tight loop that runs for a second or more and which does not perform any messaging nor check to see if the current message is closed, then you should either change the timer for the part of JActor that is executing the message or change the default.
